## 用途

已有文档结构乱、信息检索难时触发整理。

## 预检

1. 参数 `<文档路径>` 非空
2. 文件存在

## 委托

调用 Skill `managing-knowledge`，子意图 **optimize-doc**：

- 加载 `reference/optimize-doc.md` 规则
- 识别问题清单（3-5 条：结构 / 冗余 / 索引不可达等）
- 每条说明：现状、建议、影响范围
- 用户确认要优化哪几条
- 实施优化，保持核心信息不变
- 文档末尾加"最近优化记录"段
- 如锚点/标题有变：更新 INDEX.md
