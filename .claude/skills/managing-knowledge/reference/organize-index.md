# 整理目录索引

## 触发

用户说"整理 context/ 索引"，或某个 INDEX.md 内容和目录实际不符时。

## 步骤

1. **定位目标 INDEX.md**（用户指定或默认当前所在目录的）
2. **扫描同级 + 下一级文件**
3. **对比 INDEX.md 与实际文件**：
   - 实际存在但 INDEX 未列 → 遗漏
   - INDEX 列了但文件不存在 → 腐化
   - 文件重命名导致链接失效 → 腐化
4. **输出 diff 报告**
5. **用户确认后**，修正 INDEX.md

## 硬约束

- ❌ 禁止自动删除 INDEX 里的条目（可能有故意保留的"未来占位"）
- ✅ 修正前必须让用户确认
- ✅ 对每个新增/删除条目写一句理由
