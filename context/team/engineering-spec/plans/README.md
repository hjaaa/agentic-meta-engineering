## 执行原则

1. 一次只执行一个 Phase，完成后再写下一个 Phase 的 plan
2. 每个 Phase 内部按 task 顺序执行，每个 task 单独 commit
3. Phase 完成后做一次整体 review，通过才进下一 Phase
4. Phase 完成并合入 main 后，把 plan 文档 `git mv` 到 `history/`
