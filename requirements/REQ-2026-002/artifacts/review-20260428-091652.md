## Major（强烈建议修复）

### [error-handling · design-consistency] scripts/git-hooks/pre-commit:32-40

**F-01**：`set -euo pipefail` + `python3 ... run.py --trigger=pre-commit`，runner 非零退出 set -e 立即中止，line 33 `EXIT_CODE=$?` 与之后整段友好错误提示永远不执行。git commit 仍被阻断（外层 exit 1 正确传递），仅 UX 退化。

**建议**：`python3 ... || EXIT_CODE=$?; EXIT_CODE=${EXIT_CODE:-0}; if [ $EXIT_CODE -ne 0 ]; then echo '✗ ...'; fi; exit $EXIT_CODE`。或在 line 32 前 `set +e`，提示后再 `set -e`。

### [history-context] .github/workflows/quality-check.yml:5,28

**F-03**：ecfbb74 (2026-04-23) 写入的合规说明 "如需临时放行，应优先加 scripts/lib/index-config.yaml 的 ignore 白名单（需 PR review）而非关 strict" 在 adccbb5 切单 step 时被完全删除。runner 内部仍尊重 index-config.yaml（功能未变），但合规规程零可发现性。

**建议**：在 single step 上方加 5-8 行注释块重述白名单规程 + legacy escape_hatch 合规要求；或让 runner 失败 stderr 输出 hint。

### [design-consistency] scripts/check-meta.sh:24, check-index.sh:23, check-sourcing.sh:27, check-reviews.sh:19, check-plan.sh:27

**F-04**：5 个旧 .sh 末行仍是 `exec python3 "${SCRIPT_DIR}/lib/check_xxx.py" "$@"`，完全绕过 runner——无 audit log、不经 registry 校验、snapshot 行为契约对 .sh 入口无意义。注释明确声称"已由 F-002 改写为薄壳，直接 exec 统一门禁 runner（adapter 模式）"，与代码完全矛盾。features.json F-002.description + plan.md R-3 + D-001 都明确要求"旧入口在兼容期改造为薄壳直接 exec runner"。

**建议**：5 个 .sh 末行改 `exec python3 "${SCRIPT_DIR}/gates/run.py" --trigger=adapter --legacy=check-xxx "$@"`（xxx ∈ meta/index/sourcing/reviews/plan）；注释一致性顺手修。run.py 的 LEGACY_TO_PLUGIN 映射已就位。

### [complexity] scripts/gates/plugins/review_verdict.py:72

**F-05**：`_run_all_requirements` 51 行 + 圈复杂度 18（阈值 10）。单循环体内同时处理 yaml 解析异常、legacy 过滤、phase 过滤、规则聚合、findings 分类。

**建议**：拆 `_load_req_meta` + `_should_skip_req` + 主循环聚合三段，主体 CC ≤ 6。

### [error-handling] scripts/gates/run.py:260-261

**F-06**：`build_context` 读 meta.yaml 仅 `except yaml.YAMLError`，未捕获 OSError。`exists()` 后被删/权限变更，`open()` 抛 OSError 直接冒泡到 main()，无 requirement_id 上下文，traceback 暴露。

**建议**：`except (yaml.YAMLError, OSError) as exc`，加带 req_id 的 WARNING + meta 回退空 dict。

### [error-handling] scripts/gates/run.py:470

**F-07**：`_restore_state` 清理 .bak 时 `bare except Exception: pass` 静默吞错。磁盘满 / 权限问题导致 .bak 残留，无任何日志感知，后续 GATE-WORKSPACE-CLEAN 可能误报。违反 CLAUDE.md §2"不吞异常"硬规则。

**建议**：`except Exception as exc:` 内补 WARNING stderr 日志，列出 backup 路径与原因。

### [error-handling] scripts/gates/plugins/review_verdict.py:89-96

**F-08**：`_run_all_requirements` meta.yaml 解析失败 warning 仅入 `all_warnings` tuple，不打 stderr 日志。CI log 中无法 grep req_id 定位失败需求。违反 CLAUDE.md §3.2"日志必须包含业务主键"。

**建议**：追加 tuple 前先 `print(f'WARNING GATE-REVIEW-VERDICT req={req_id} meta.yaml 解析失败: {exc}', file=sys.stderr)`。

### [error-handling · security] scripts/gates/run.py:426

**F-09**：plugin 异常时 `traceback.print_exc(file=sys.stderr)` 暴露内部实现到 CI stderr。GitHub Actions 把 stderr 完整 expose 到 PR 评论 / workflow log；堆栈泄漏内部文件路径与函数名。违反 §9 安全规范。

**建议**：保留 traceback 写 audit log；CI 环境（`os.environ.get('CI')`）下抑制 print_exc，仅打 `ERROR plugin 执行异常 plugin=... trigger=... req=... : <exc>`。
