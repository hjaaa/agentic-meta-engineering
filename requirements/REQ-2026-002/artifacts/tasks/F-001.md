# F-001 · Runner 骨架 + Gate 基类 + registry schema + 2 个 adapter

## 需求描述

搭建 `scripts/gates/` 目录骨架；实现 Gate 抽象基类、GateContext / Report / Severity / Skip 数据类、registry.yaml 加载器（含 S1~S10 schema 校验 + graphlib 依赖图无环校验）、run.py 主入口（trigger 过滤 + 拓扑排序 + 事务包裹 + audit log 写入）、plugins/meta_schema.py + plugins/index_integrity.py 两个 adapter（逻辑零改动）、migration/normalize-stderr.sh、`make gates-validate` 入口。旧 check-meta.sh / check-index.sh 暂不改动。

## 接口/数据结构

详见 `requirements/REQ-2026-002/artifacts/detailed-design.md`：
- §1 registry.yaml 字段精确定义（S1~S10）
- §2.1 plugins/base.py 接口签名（Gate / GateContext / Report / Severity / Skip）
- §2.2 run.py 主入口伪代码（含 commit_staged_writes 时序）
- §2.3 audit JSON schema
- §5.1 migration/normalize-stderr.sh

## 验收

详见 `features.json` F-001.acceptance：
1. `make gates-validate` 通过：registry schema + 依赖图 + 所有 plugin 加载
2. `python scripts/gates/run.py --trigger=ci --dry-run` 输出 Report 含 `GATE-META-SCHEMA` + `GATE-INDEX-INTEGRITY`
3. snapshot 对比：旧 `check-meta.sh` vs `run.py --trigger=adapter --legacy=check-meta`，归一化后 stderr 关键前缀行 diff 为空
4. 覆盖 pass / fail / skip 三种用例的 plugin 测试全过
