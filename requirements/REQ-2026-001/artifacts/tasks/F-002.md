# F-002 · Worker 后台脚本：截断 + claude -p + notes.md 写入

## 需求描述

新建 `.claude/hooks/extract-experience-worker.sh`，实现 30s watchdog、`tail -n 60` 截断、claude -p 调用（`--add-dir` + `</dev/null` 关 stdin 绕 #7263）、`mkdir` 原子锁、notes.md 追加写入（成功小节 / 失败 `[hook-skipped]` 标记）。worker 内部对 `claude` 命令支持 `EXPERIENCE_HOOK_CLAUDE_BIN` 环境变量覆盖以便 mock 测试。

## 接口/数据结构

详见 `requirements/REQ-2026-001/artifacts/detailed-design.md` 第 4 节（M3 Worker 完整伪代码）。
- 入参：`$1=CWD $2=TRANSCRIPT_PATH $3=NOTES_FILE`
- notes.md 写入格式：`## 会话经验（YYYY-MM-DD HH:MM）\n\n<内容>` 或 `_[hook-skipped: <原因>]_`
- 错误码白名单：`transcript-truncate-failed` / `prompt-template-missing` / `claude-exit-N` / `empty-output-bug-7263` / `notes-busy`

## 开发笔记

- worker 第一行必须 `cd "$CWD"`（修 R-9：worker 内部调 git 等命令时 CWD 已不是会话 cwd）
- watchdog `(sleep 30 && kill -9 $$) &` 必须在 trap 之前启动，且 trap 里要 `kill watchdog_pid`
- 调用 `claude -p` 时显式 `SKIP_EXPERIENCE_HOOK=1` 短路递归（防 R-5）
- 调用 `claude -p` 必须 `</dev/null` 关 stdin（绕 #7263 大 stdin 空输出）
- mkdir 锁 5 次重试，每次 sleep 0.1
- worker 内部所有 claude 调用用 `"${EXPERIENCE_HOOK_CLAUDE_BIN:-claude}"`，便于 F-004 mock 测试
